# ClawdView Implementation Patterns

Our Professional Services organization standardizes deployment into modular phases to ensure time-to-value. 

## Foundational Timelines
- **[[clawdview-agileplace]]:** Fast deployment. Typically 30–45 days to configure boards and map value streams.
- **[[clawdview-ppm-pro]]:** Structured deployment. 60–90 days encompassing intake routing, capacity planning, and user training.
- **[[clawdview-portfolios]]:** Transformational deployment. 90–120+ days due to the need to align financial teams, define strategic themes, and train the EPMO.

## The 'Hub' Risk Factor
When [[clawdview-hub]] is included in an SOW, the SLA requires an additional 30 days of buffer.
Hub implementations are highly dependent on the client's internal IT API maturity and the state of their legacy systems.

## Key CSM Directives
1. Always secure executive sponsorship early. If a Portfolios deployment loses the CFO, the project stalls.
2. Push for MVP integrations with Hub. Do not attempt to map every single custom Jira field on Day 1. Start with Status, Priority, and Assignee.
