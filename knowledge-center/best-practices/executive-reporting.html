Executive Dashboard Setup

Build executive-ready dashboards that communicate portfolio health, risk exposure, and strategic alignment — in under 30 seconds of attention.

Executive Dashboard Principles

Executive dashboards fail when they overload with operational detail. A CIO doesn't need to see individual task status — they need to know: "Are we going to hit our strategic objectives this quarter?" Design with these principles:

  • 30-Second Rule: An executive should grasp the portfolio state in 30 seconds
  • Exception-Based: Highlight what needs attention, not what's going well
  • Actionable: Every metric should have a "so what?" — what decision does this inform?
  • Drillable: Allow drill-down for the curious, but don't require it for comprehension

Recommended Dashboard Layout

Top Row: Portfolio Health Score

A single composite metric (0-100) that aggregates schedule, budget, risk, and scope indicators. Configure in Portfolios → Dashboard → Health Score Widget. Recommended formula:

  • Schedule Performance Index (SPI ≥ 0.9 = healthy): 30% weight
  • Budget Variance (< 10% overrun=healthy): 25% weight
  • Critical Risk Count (0 = healthy): 25% weight
  • Scope Change Rate (< 15%=healthy): 20% weight

Middle Row: Strategic Alignment Matrix

A bubble chart showing initiatives plotted against strategic themes (x-axis) and investment level (y-axis). Bubble size = team count. Color = health status. This view immediately shows if spending aligns with stated priorities.

Bottom Row: Key Metrics Trend

Sparkline charts showing 6-month trends for: portfolio ROI, resource utilization, schedule predictability, and customer satisfaction. Trend direction matters more than absolute numbers at the executive level.

💡 Configuration Tip

Set dashboards to auto-refresh every 15 minutes on Enterprise tier. For Standard tier, schedule a daily build at 7 AM local time. Avoid real-time refresh for executive dashboards — data stability matters more than currency at this level. Metric fluctuations throughout the day create unnecessary anxiety.

Data Quality Requirements

Dashboards are only as good as the underlying data. Before rolling out executive dashboards, ensure:

  • All active projects have current status updates (< 7 days old)
  • Budget data is synced from ERP via Hub (not manually entered)
  • Risk registers are maintained with probability and impact scores
  • Resource allocations reflect actual assignments (not last quarter's plan)
⚠ Anti-Pattern: "Watermelon" Projects

Projects that show green on the dashboard but are actually deep in trouble (red underneath). The #1 cause: status updates based on self-reporting rather than objective data. Solution: use PPM Pro's earned value metrics (CPI, SPI) which are computed from actual schedule and cost data, eliminating subjective bias.
