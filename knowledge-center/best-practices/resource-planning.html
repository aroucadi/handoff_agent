Cross-Project Resource Planning

Balance resource capacity across projects, teams, and time horizons using ClawdView PPM Pro's resource management module. Avoid over-allocation, skill gaps, and delivery delays.

The Resource Planning Challenge

Resource management is consistently rated as the #1 challenge for PMOs according to PMI and Gartner surveys. The problem compounds as organizations grow: 100 people across 20 projects creates 2,000 potential allocation decisions — impossible to manage in spreadsheets.

Three-Horizon Planning Model

Horizon 1: Strategic Capacity (12+ months)

Use Portfolios → Capacity Planning to model role-based demand vs. supply. Questions answered:

  • Do we have enough cloud architects for next year's digital transformation initiatives?
  • Which skills will we need that we don't currently have?
  • Should we hire, outsource, or reskill to fill gaps?

Horizon 2: Tactical Allocation (1-3 months)

Use PPM Pro → Resource Manager to assign named resources to project tasks. Key features:

  • Capacity Heat Map: Color-coded view showing utilization across all resources. Green (70-85%), Yellow (85-95%), Red (>95%).
  • Smart Suggest: AI-powered resource recommendations based on skill match, availability, cost rate, and location.
  • What-If Scenarios: Model the impact of adding/removing a project before committing resources.

Horizon 3: Operational Tracking (Daily/Weekly)

Use PPM Pro → Timesheets to capture actual effort. Compare planned vs. actual utilization to calibrate future estimates.

⚠ Common Mistake

Never plan resources at 100% utilization. This leaves zero buffer for meetings, admin, support interruptions, and unplanned work. Best practice targets: Individual contributors: 80% billable, Managers: 60%, Executives: 30%. Build these targets into your PPM Pro role configuration.

Skill-Based vs. Named Resource Planning

Approach When to Use ClawdView Feature
Role-based (generic) Early planning, 3+ months out Portfolios Capacity Planning
Skill-based (filtered) Project initiation, 1-3 months out PPM Pro Smart Suggest with skill filters
Named resource Execution, current/next sprint PPM Pro Resource Assignments

Resource Optimization Techniques

  • Resource Leveling: PPM Pro automatically adjusts task dates to resolve over-allocation while respecting dependencies. Use cautiously — it may push project end dates.
  • Resource Smoothing: Optimizes resource usage within existing project constraints — prevents end-date changes but may not resolve all conflicts.
  • T-Shaped Resourcing: Build teams with deep specialists and broad generalists. Track primary + secondary skills in the resource profile to enable cross-training assignments.
  • Shared Resource Pools: For scarce specialties (DBAs, security engineers, UX researchers), create a shared pool managed by a resource manager. Projects request allocation through a booking system rather than direct assignment.
💡 Best Practice

Review resource allocation at two cadences: weekly team-level (are people over/under-utilized this sprint?) and monthly portfolio-level (are we investing our best people in our highest-priority initiatives?). Use the PPM Pro Utilization Dashboard for weekly reviews and Portfolios Capacity View for monthly reviews.
