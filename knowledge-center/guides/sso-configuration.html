SSO & SAML Configuration

Configure single sign-on for ClawdView using Okta, Azure AD, OneLogin, Google Workspace, or any SAML 2.0 / OIDC-compatible identity provider.

Supported Protocols

Protocol Status Recommended For
SAML 2.0 ✅ Fully supported Enterprise IdPs (Okta, Azure AD, ADFS)
OpenID Connect (OIDC) ✅ Fully supported Google Workspace, Auth0, Keycloak
LDAP/AD Direct ⚠ Via connector only On-premise AD without federation

Configuration Steps

Step 1: Prepare Your IdP

In your identity provider, create a new SAML 2.0 application with these settings:

  • Entity ID: https://auth.clawdview.com/saml/{your-tenant-slug}
  • ACS URL: https://auth.clawdview.com/saml/{your-tenant-slug}/acs
  • Name ID Format: emailAddress (required)
  • Signed Response: Required (SHA-256)

Step 2: Configure Attribute Mapping

Map the following IdP attributes to ClawdView user properties:

ClawdView Property Required Okta Attribute Azure AD Attribute
email ✅ Yes user.email user.mail
firstName ✅ Yes user.firstName user.givenname
lastName ✅ Yes user.lastName user.surname
department Optional user.department user.department
groups Optional user.groups user.assignedroles

Step 3: Enable in ClawdView Admin

Navigate to Admin → Security → SSO Configuration:

  1. Upload your IdP's metadata XML file (or paste the metadata URL)
  2. ClawdView auto-populates the IdP Entity ID, SSO URL, and certificate
  3. Test the connection with the "Test SSO" button (pre-production validation)
  4. Enable SSO for your tenant. Optionally enforce SSO-only login (disables password auth)
⚠ Important

Before enforcing SSO-only login, ensure at least one emergency admin account retains password access. If your IdP goes down, this account is the only way to access ClawdView and disable the SSO enforcement. Store the credentials in a secure vault (e.g., 1Password, HashiCorp Vault).

Group-Based Provisioning

For organizations with 200+ users, manual user management is impractical. ClawdView supports SCIM 2.0 provisioning for automated user lifecycle management:

  • Auto-Create: When a user is added to the ClawdView group in the IdP, their ClawdView account is automatically created
  • Auto-Deactivate: When removed from the IdP group, the ClawdView account is deactivated (data preserved for 90 days)
  • Group → Role Mapping: Map IdP groups to ClawdView roles (e.g., "CV-Admins" → Admin role, "CV-PMO" → PMO role)
  • Group → Workspace Mapping: Map IdP groups to ClawdView workspaces for automatic access provisioning

Troubleshooting

Issue Cause Solution
SAML response signature invalid Certificate mismatch or clock skew Re-upload IdP certificate; ensure NTP sync (±5 min tolerance)
User not found after SSO Email attribute not mapped Verify NameID format is emailAddress in IdP config
Groups not syncing Group attribute not released Add group claim to SAML assertion in IdP
Redirect loop ACS URL mismatch Verify tenant slug matches exactly in both IdP and ClawdView
