API Integration Patterns

REST and GraphQL API reference for ClawdView, covering authentication, endpoints, webhooks, rate limits, pagination, and SDK usage for building custom integrations.

Authentication

ClawdView APIs support two authentication methods:

API Key (Service Integrations)

Generate API keys in Admin → API Keys. Each key is scoped to a specific workspace and has configurable permissions (read, write, admin). Keys are transmitted via the X-API-Key header.

curl -H "X-API-Key: cv_live_abc123..." https://api.clawdview.com/v2/projects

OAuth 2.0 (User-Context Integrations)

For integrations that act on behalf of users, use the OAuth 2.0 authorization code flow. ClawdView acts as the authorization server with JWT access tokens (15-minute expiry) and refresh tokens (30-day expiry).

Rate Limits

Tier Requests/Minute Burst Limit Concurrent
Standard 100 20 req/sec 5
Enterprise 1,000 50 req/sec 20
Partner 5,000 100 req/sec 50

Rate limit headers are returned on every response: X-RateLimit-Remaining, X-RateLimit-Reset. When limits are exceeded, the API returns 429 Too Many Requests with a Retry-After header.

Pagination

List endpoints return paginated results using cursor-based pagination:

GET /v2/projects?limit=50&cursor=eyJpZCI6MTIzfQ== Response: { "data": [...], "pagination": { "next_cursor": "eyJpZCI6MTczfQ==", "has_more": true, "total_count": 342 } }

Webhooks

ClawdView can send real-time notifications to your endpoints when resources change:

  • Events: project.created, project.updated, task.completed, card.moved, budget.exceeded, risk.escalated
  • Delivery: HTTPS POST with JSON payload. HMAC-SHA256 signature in X-CV-Signature header for verification.
  • Retry Policy: 3 attempts with exponential backoff (1s, 30s, 5min). Failed delivery after 3 attempts → event logged to webhook failure dashboard.
  • Filtering: Configure per-webhook event types and resource scopes (e.g., only project.updated for workspace "IT PMO")
💡 Webhook Best Practice

Always verify the X-CV-Signature header before processing webhook payloads. Use a shared secret configured in Admin → Webhooks. The signature is computed as HMAC-SHA256(secret, payload_body). Failing to verify signatures exposes your endpoint to spoofed events.

GraphQL API (Beta)

For complex queries that span multiple resources, ClawdView offers a GraphQL endpoint at https://api.clawdview.com/graphql. This is particularly useful for:

  • Fetching a project with its tasks, resources, and financials in a single request
  • Querying portfolio items with filtered scoring criteria
  • Real-time subscriptions for board card movement (WebSocket-based)
⚠ Beta Notice

The GraphQL API is in public beta. Breaking changes may occur between releases. Pin to specific schema versions using the X-Schema-Version header. GA is planned for v4.3.0.
